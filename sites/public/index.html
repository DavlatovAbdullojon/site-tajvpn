Как это работает

Автоматический платежный поток

Мобильное приложение создаёт заказ, открывает страницу оплаты ENOT и затем опрашивает статус. После подтверждения платежа доступ к VPN активируется без ручной модерации чеков.

Страницы callback

Готово для success / fail

Сайт уже содержит страницы для /payment/success и /payment/fail, чтобы их можно было сразу указать в настройках ENOT.

Backend health

Минимум ручной рутины

Backend отдаёт тарифы, создаёт инвойсы, принимает webhook ENOT, считает подписку и отдаёт admin API с токеном в заголовке X-Admin-Token.
